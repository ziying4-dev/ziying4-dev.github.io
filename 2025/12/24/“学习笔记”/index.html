Hexo学习笔记

1.本地启动指令

  • hexo s:启动本地服务器

  • hexo g:生成静态文件

2.新建文章

执行hexo new “文章标题”就能创建新的Markdown文件。

PS:1.冒号后必须加一个空格：如果是多个标签，写成[“编程”，”Hexo”,”学习’’]

​ 2.头信息引号是英文

​ 3.头信息和正文之间要留一个空行

3.补充内容

步骤 1：打开 Git Bash 终端

1.双击桌面的Git Bash图标 ；
2.终端打开后，执行命令进入 Hexo 博客目录：

cd /c/Users/Lenovo/Desktop/hexo-blog

3.执行ls命令，能看到_config.yml、source等文件夹，说明路径正确。

步骤 2：打开 Typora 编辑现有笔记

1.点击 桌面 > hexo-blog > source > _posts；
2.找到学习笔记.md文件，右键选择打开方式 > Typora；
3.在 Typora 中直接在现有内容后续写笔记（按 Markdown 语法编写），写完按Ctrl+S保存文件。

步骤 3：重新生成并部署博客

回到Git Bash 终端，执行以下命令更新博客并推送到 GitHub Pages：

hexo clean # 清理旧的静态文件
hexo g # 生成包含新内容的静态文件
hexo d # 部署到GitHub仓库

若弹出 GitHub 登录验证，按提示完成授权即可。

步骤 4：验证更新结果

稍等 1-2 分钟，打开浏览器访问https://ziying4-dev.github.io，能看到学习笔记里新增的内容，说明更新成功。
